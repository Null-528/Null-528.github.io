1.3 进程和线程的区别：

  1. 根本区别：进程是操作系统资源分配的基本单位，线程是CPU调度的基本单位。
  2. 资源方面，进程有自己的独立地址空间，是拥有系统资源的一个独立单位，而线程共享所属进程的地址空间，自己基本上不拥有系统资源，只拥有一点在运行中必不可少的资源(如程序计数器,一组寄存器和栈)，和其他线程共享本进程的相关资源如内存、I/O、cpu等；
  3. 数量方面，线程依赖于进程而存在，一个进程至少有一个线程；
  4. 开销方面，在进程切换时，涉及到整个当前进程CPU环境的保存环境的设置以及新被调度运行的CPU环境的设置，而线程切换只需保存和设置少量的寄存器的内容，并不涉及存储器管理方面的操作，可见，进程切换的开销远大于线程切换的开销；
  5. 通信方面，由于同一进程中的多个线程具有相同的地址空间，致使它们之间的同步与通信的实现也变得比较容易，而进程之间的通信需要以进程间通信(IPC)的方式进行；

    同一进程中的线程可以共享哪些数据？

    1. 进程的公有数据（全局变量、静态变量…）
    2. 进程打开的文件描述符
    3. 进程的当前目录
    4. 信号处理器/信号处理函数：对收到的信号的处理方式
    5. 进程ID与进程组ID

线程独占哪些资源？

  1. 线程ID
  2. 一组寄存器的值
  3. 线程自身的栈（堆是共享的）
  4. 错误返回码：线程可能会产生不同的错误返回码，一个线程的错误返回码不应该被其它线程修改；
  5. 信号掩码/信号屏蔽字(Signal mask)：表示是否屏蔽/阻塞相应的信号（SIGKILL,SIGSTOP除外）

进程间通信有哪些方式？

  1. 管道(Pipe)

    管道是半双工的，数据只能向一个方向流动；需要双方通信时，需要建立起两个管道；
    一个进程向管道中写的内容被管道另一端的进程读出。写入的内容每次都添加在管道缓冲区的末尾，并且每次都是从缓冲区的头部读出数据；
    只能用于父子进程或者兄弟进程之间(具有亲缘关系的进程)

  2. 命名管道
  3. 消息队列
  4. 信号(Signal)
  5. 共享内存
  6. 信号量(Semaphore)：初始化操作、P操作、V操作；P操作：信号量-1，检测是否小于0，小于则进程进入阻塞状态；V操作：信号量+1，若小于等于0，则从队列中唤醒一个等待的进程进入就绪态
  7. 套接字(Socket)

2. 多线程

在一个进程中运行了多个线程，便是多线程。目的是避免因为 IO 阻塞而导致的线程等待，从而提高程序的执行效率和 CPU 利用率。
同时也要注意，线程并不是越多越好，如果线程过多造成频繁切换进程上下文，线程从 waiting 切换 runnable 都需要进行上下文切换，反而降低执行效率。

减少上下文切换的办法：

  1. 无锁并发编程。多线程竞争锁时会引起上下文切换，使用 Hash环，将数据 id 取模分段，每个线程处理不同数据，可以大大降低锁的竞争。
  2. CAS，如 java 的 atomic 包使用 CAS 算法来更新数据，避免了锁的开销。
  3. 使用最少线程：避免创建不需要的线程，比如任务很少，但是创建了很多线程waiting，反而造成频繁切换进程上下文，降低效率。
  4. 协程：在单线程里实现多任务的调度，并在单线程里维持多个任务间的切换，可以避免多线程竞争锁的开销。

2.1 串行、并发、并行

  1. 串行：一个任务执行完毕后，再执行下一个
  2. 并发：多个任务交替执行，宏观上看是同时执行的。并发的多个任务之间是互相抢占资源的。
  3. 并行：多个任务同时执行，只有在多CPU的情况中，才会发生并行。

    并行是多任务同时执行，并发是多任务交替执行。并行是多核CPU的特性，并发是多任务的特性。并行是并发 ps的子集。

2.2 同步和异步、阻塞和非阻塞

  1. 同步：执行某个功能后，被调用者不会主动同住返回结果。
  2. 异步：执行某个功能后，被调用者主动通知返回结果。

    同步和异步的区别在于是否需要等待被调用者返回结果。

  3. 阻塞：当一个线程在执行一个耗时的IO操作（如网络请求、文件读写等）时，它会暂停在该位置，直到该操作完成或者超时后才返回结果。
  4. 非阻塞：当一个线程在进行一个耗时的IO操作时，它不会停下来等待这个操作的结果，而是立即返回一个状态值表示正在进行中，然后可以继续执行其他的代码。

    阻塞和非阻塞的区别在于是否在某个操作完成时，当前线程可以继续执行其他任务。

3 线程的创建与常见方法

3.1 线程的创建

  1. 继承 Thread 类，重写 run 方法。但由于 java 单继承，因此不推荐。

  2. 实现 Runnable 接口，重写 run 方法，然后 Thread 对象来启动线程，比1更灵活。

    注意：启动线程要用 start 方法，会创建一个新线程，而不是直接调用 run 方法，run 方法只是普通方法，不会创建新线程。
    另外，以上两种方法由于 run() 返回值为 void，不适合用于需要携带返回结果的线程

  3. 实现 Callable 接口，重写 call 方法，配合 FutureTask 包装 Callable 实现类，再通过 Thread 来启动线程(同步非阻塞）。

  4. 使用线程池（ExecutorService）

  5. 使用线程池（ExecutorService）

  6. 使用 FutureTask

  7. 使用并发工具类，如 CountDownLatch、CyclicBarrier 等。

  8. 使用并发框架中的其他组件，例如 CompletableFuture、ForkJoinPool 等。

3.2.2 守护线程

默认情况下，线程都是非守护线程。守护线程是一种支持型线程，主要用于程序中后台调度以及支持性工作。常见线程及性质：

  1. 主线程：默认是非守护线程，如果主线程执行结束，需要查看当前JVM内是否还有非守护线程，如果没有JVM直接停止。
  2. GC执行Object的finalize方法：执行这个方法的线程就是守护线程。
    将非守护线程设置为守护线程： 另外注意：
  3. 守护线程中产生的新线程也是守护线程。
  4. 守护线程不能持有非守护线程的资源，否则会导致 JVM 无法退出。
  5. 不要依靠守护线程中的 finally 清理资源，如果没有非守护线程，JVM 直接退出，这时候会导致守护线程的资源不会被清理。

3.2.3 wait 和 notify

首先，wait 会释放锁，sleep 则不会。因此由于涉及锁操作，如果使用了 synchronize 锁，那么wait() 和 notify() 、notifyAll() 方法都要在 synchronize 代码块中使用。

3.2.4 线程的结束

  1. return 结束
  2. 抛出异常结束
  3. 直接调用 stop 方法结束，不推荐使用。他会强制线程结束，不论当前线程执行到哪里。
  4. volatile 共享变量，当共享变量满足某个条件时，结束线程。
  5. 使用 interrupt 方法结束，通过设置中断标志位来终止线程。
